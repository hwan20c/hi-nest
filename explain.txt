npm run test:cov
-> 모든 spec.ts 파일을 찾아서 테스트를 해준다.

npm run test:watch
-> 모든 ts 파일을 찾아준다 그 후에 옵션을 주면서 작동시킬 수 있다.

e2e 테스트
-> 개발자가 아닌 사용자가 움직이는 부분들에 대해서 처음부터 끝까지 테스트하게 할 수 있는 것

------------------------------------------------------

e2e 테스트는 테스트할때마다 어플리케이션을 생성한다.
테스트에서 생성되는 어플리케이션과 기본적으로 실행할때 사용되는 어플리케이션은 완전히 다른것이다. 

실제로 사용하는 서버에서는 number로 되어있어도 작동하는데
e2e에서는 string으로 받아버려서 작동을 하지 않게 된다.
mian.ts 에서 transform을 해주지만 테스트에서는 해주지 않기 때문이다.
